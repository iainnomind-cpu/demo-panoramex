# Implementation Plan: Módulo de Campañas de WhatsApp

**Branch**: `[005-campanas-whatsapp]` | **Date**: 2026-07-03 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/005-campanas-whatsapp/spec.md` y directiva "Reutilizar el motor de envío/plantillas construido en la fase 3. Nueva tabla campaigns y campaign_sends en Supabase. Un Vercel Cron Job (definido en vercel.json) invoca diariamente una función serverless que revisa cumpleaños del día y dispara la campaña correspondiente. satisfaction_surveys como tabla separada, vinculada a reservations."

## Summary

Implementar un motor de campañas masivas de WhatsApp, encuestas de satisfacción y automatizaciones de cumpleaños. Se reutilizará la lógica de envío de mensajes y plantillas (desarrollada en la Fase 3 del Chatbot), apoyada por nuevas tablas (`campaigns`, `campaign_sends`, `satisfaction_surveys`). Se usará `vercel.json` con `crons` para ejecutar revisiones diarias. El parseo de CSVs se hará en el frontend para mandar lotes al backend y sortear los timeouts y límites de payload.

## Technical Context

**Language/Version**: TypeScript 5+

**Primary Dependencies**: React 18, Vite, Tailwind CSS, Zustand, Supabase (Auth/DB/Realtime), Vercel Serverless Functions, PapaParse (para CSV).

**Storage**: PostgreSQL (Supabase).

**Testing**: Pruebas manuales estructuradas (quickstart) y reglas de negocio robustas en DB (RLS).

**Target Platform**: Web (Frontend) y Vercel (Backend API + Cron).

**Project Type**: Web Application CRM.

**Performance Goals**: Despachar 5,000 mensajes en <10 minutos. Importación de CSV de 10,000 filas.

**Constraints**: Throttling de Meta API, Timeouts de Vercel (generalmente 10s-60s), Payload limits (4.5MB en Vercel). Sin API keys en el cliente. RLS obligatorio.

**Scale/Scope**: Operadores de Panoramex, miles de prospectos, encuestas recurrentes y 1 evento cron por día (o varios según configuración).
