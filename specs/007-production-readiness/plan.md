# Implementation Plan: Preparación para Producción

**Branch**: `007-production-readiness` | **Date**: 2026-07-03 | **Spec**: [spec.md](file:///c:/Users/redi_/Downloads/panoramex-crm/specs/007-production-readiness/spec.md)

## Summary

Preparación final del CRM para uso en producción por parte de Panoramex, abarcando experiencia instalable en móvil (PWA con vite-plugin-pwa), administración central de pausa global (Supabase DB config), control de acceso por rate limiting a webhooks (Vercel Edge middleware / in-memory), exportación completa de datos de organización y verificación de Backups.

## Technical Context

**Language/Version**: TypeScript 5+ (React 18, Node.js para Vercel)

**Primary Dependencies**: Vite, `vite-plugin-pwa` (para manifiesto y Service Worker), Supabase Client.

**Storage**: PostgreSQL (vía Supabase)

**Testing**: N/A formal, pruebas manuales definidas en quickstart.

**Target Platform**: Web y dispositivos móviles iOS/Android (via PWA)

**Project Type**: Web Application & Serverless Backend

**Performance Goals**: Exportación de datos en menos de 60s. Rate Limiter respondiendo en <50ms.

**Constraints**: Respetar cuotas de Vercel (timeout 10-60s). Offline capabilities limitadas a UI básica (no sincronización compleja para MVP).

**Scale/Scope**: 7 usuarios internos (colaboradores), datos crecientes en Supabase.
