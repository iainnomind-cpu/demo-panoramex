## Anti-Patterns Verdict

**LLM Assessment**: The Dashboard does not immediately read as "AI slop" — it avoids the most egregious tells. There is no gradient text, no glassmorphism, no decorative grid overlays, no side-stripe borders, no numbered section markers. The TopTours image grid with gradient overlay shows genuine visual ambition. However, two second-order tells remain:

1. **The identical-weight KPI grid**: 4 KPI cards in a 2×2 / 4-column grid, each with an icon container + big number + label + trend indicator. This is the "hero-metric template" pattern — big number, small label, supporting stat. It doesn't have the gradient accent, but the structural DNA is the same SaaS cliché. Nothing differentiates Leads Hoy from Conversión visually, even though their importance is radically different.

2. **Color token drift**: Despite having a committed design system (DESIGN.md, tailwind.config.ts), the Dashboard components ignore it. `bg-white`, `border-gray-200`, `text-gray-900`, `text-gray-500` — this is the Tailwind default stylesheet, not a Panoramex CRM. The nav sidebar uses brand tokens correctly; the content area doesn't.

**Deterministic Scan**: 2 advisory findings in `SalesFunnel.tsx`:
- Line 47: `#374151` (Tailwind gray-700) used in `tick={{ fill: '#374151' }}` — undocumented color outside DESIGN.md palette
- Line 52: `borderRadius: 10px` in Tooltip `contentStyle` — outside DESIGN.md rounded scale (`sm:4px md:8px lg:12px xl:16px`)
Both are low-severity drift, not blocking issues. No false positives detected.

## Overall Impression

The Dashboard has good structural bones — a clear 3-row hierarchy (KPIs → Funnel+Activity → Tours), a working semaphore badge system, and a visually strong tour image grid. The fundamental information architecture is sound. But the content area looks like a different product from the sidebar. The navy sidebar is Panoramex CRM; the white content area with gray Tailwind defaults is Generic React Dashboard. Closing that gap is the single biggest opportunity.

## What's Working

1. **The Semaphore Badge System**: Status badges in RecentActivity are exactly right — pill shape, tinted background, matching-hue border, icon prefix, semaphore colors. An agent can read prospect status in under a second, which directly serves the "Speed is a feature" principle.

2. **The TopTours Image Grid**: `aspect-[4/3]` cards with `group-hover:scale-105` zoom, `bg-gradient-to-t from-black/70` text overlay, and price/reservas data creates a genuinely premium moment. This is the only section that looks like the "modern, sleek, premium" brand personality.

3. **Sidebar Navigation**: The deep navy sidebar with active coral text is on-brand and clear. Nav items are labeled (not icon-only), active state is unambiguous, and the structure is immediately learnable.

## Priority Issues

**[P1] Design Token Abandonment in All Content Components**
- **What**: KPICard, SalesFunnel, RecentActivity, and TopTours all use raw Tailwind `gray-*` classes instead of the DESIGN.md token system. `bg-white`, `border-gray-200`, `text-gray-900`, `text-gray-500`, `text-gray-400` appear throughout — none of these are tokens.
- **Why it matters**: The sidebar and content area look like different products. A sales agent spending 6-8 hours/day in this tool will unconsciously feel the dissonance even if they can't name it. More practically: if the theme ever changes (dark mode, client white-labeling), the content area won't update.
- **Fix**: Replace `bg-white` → `bg-surface-container-lowest`, `border-gray-200` → `border-outline-variant`, `text-gray-900` → `text-on-surface`, `text-gray-500` → `text-on-surface-variant`, `text-gray-400` → `text-outline`. Four search-and-replace operations.
- **Suggested command**: `$impeccable polish Dashboard`

**[P1] Wrong Border Radius on Cards (Modal Territory)**
- **What**: KPICard uses `rounded-2xl`, SalesFunnel uses `rounded-2xl`, RecentActivity uses `rounded-2xl`, TopTours uses `rounded-2xl`. DESIGN.md reserves `rounded-2xl` (24px) exclusively for modals. Cards use `rounded-xl` (16px).
- **Why it matters**: Over-rounded cards are a documented Codex tell ("picking 24/28/32/40px on a card is the codex tell; no brand wants 'insanely rounded'"). It also makes content cards feel as elevated as modal dialogs, flattening the visual hierarchy.
- **Fix**: Change all four components from `rounded-2xl` to `rounded-xl`.
- **Suggested command**: `$impeccable polish Dashboard`

**[P1] No Loading or Empty States on Any Component**
- **What**: All four Dashboard components render whatever data is available with zero skeleton states, zero empty state messages, and zero error boundaries. RecentActivity with 0 prospects shows a white box with a scrollbar. KPI values show whatever the store returns with no loading indicator.
- **Why it matters**: Cianya (the sales agent) opens the Dashboard every morning. If data is loading or the connection is slow, she sees numbers flickering in or empty boxes. For a tool that must feel premium, this is a trust-breaker.
- **Fix**: Add skeleton shimmer states to KPICard (the `.shimmer` utility is already defined in `index.css`). Add a "Sin prospectos recientes" empty state to RecentActivity with a CTA to `/prospectos`. Add a loading prop path to SalesFunnel.
- **Suggested command**: `$impeccable harden Dashboard`

**[P2] Hardcoded Fake Data in TopTours**
- **What**: `const sales = [12, 8, 5, 3]` is hardcoded on line 10 of TopTours.tsx. These "reservas" numbers are invented, not from the data layer.
- **Why it matters**: An agent checking tour performance will see numbers that don't reflect reality. This is exactly the kind of subtle wrongness that erodes trust in a CRM — "if the numbers are wrong here, where else are they wrong?"
- **Fix**: Either derive `sales` from actual reservation data in the store, or remove the field entirely until real data is available. Do not display a metric you cannot source.
- **Suggested command**: `$impeccable harden Dashboard`

**[P2] Non-Functional Filter Button in SalesFunnel**
- **What**: The `filter_alt` icon button in SalesFunnel has no `onClick` handler. It renders as a tappable element that does nothing.
- **Why it matters**: Dead UI controls destroy trust faster than missing UI controls. An agent who clicks "filter" and nothing happens will assume the app is broken.
- **Fix**: Either wire a real filter (date range, status filter) or remove the button until it's implemented. A ghost element is worse than no element.
- **Suggested command**: `$impeccable harden Dashboard`

## Persona Red Flags

**Alex (Power User)**
- No keyboard navigation on the prospect rows in RecentActivity — must click, cannot Tab and Enter
- No keyboard shortcut to jump to any module (e.g. `G + P` for Prospectos, `G + R` for Reservaciones — Notion/Linear style)
- No bulk actions: cannot select multiple KPI cards or prospects
- SalesFunnel filter button is a dead control — actively hostile to power users who reach for it
- The entire Dashboard is read-only: Alex cannot update a prospect status, log a call, or book a tour from here. Must navigate away for every action.

**Sam (Accessibility-Dependent)**
- Clickable `div` in RecentActivity has no `role="button"`, no `tabIndex={0}`, no `onKeyDown` handler — completely inaccessible by keyboard
- KPI trend indicators use color alone (emerald = up, red = down) with no accessible text alternative for the trend direction
- TopTours clickable cards (`div` with `onClick`) have no `role`, `tabIndex`, or keyboard handler
- Filter button in SalesFunnel has no `aria-label` — screen reader announces it as an unlabeled button
- `text-gray-400` (#9ca3af on white) fails WCAG AA contrast (3.4:1, requires 4.5:1) — used for subtitle text in KPICard

**Cianya (Sales Agent, 6-8hrs/day)**
- Dashboard is entirely passive — she can see status but cannot act on it. Every action requires navigation. After checking the Dashboard, she must go to Prospectos to call, to Reservaciones to book, to Conversaciones to message. No quick actions from the most-used view.
- The 4 KPI cards show the same visual weight — there's no way to tell that "Conversión 12%" is critical vs "Leads Hoy 24" is informational. Under quota pressure, she needs priority signals, not equal-weight metrics.
- RecentActivity only shows 4 prospects maximum, hardcoded with `slice(0, 4)`. If she manages 30-50 active prospects, this panel becomes noise rather than signal — she can't configure what "recent" means.

## Minor Observations

- `font-mono` class on the phone number in RecentActivity (line 47) is correct behavior but uses the class name rather than the `font-mono` token — fine, but worth making explicit
- KPI card icon containers use ad-hoc Tailwind colors (`bg-blue-50 text-blue-600`, `bg-emerald-50 text-emerald-600`) — these are analytical role colors as documented in DESIGN.md, but they're not in the token file; consider adding `analytical-*` tokens
- SalesFunnel `#374151` (detected) and `borderRadius: 10px` are minor token drift in the Recharts internals — not worth a full pass but should be tidied
- TopTours image fallback URL is a hardcoded Unsplash URL with an `auto=format` parameter — this will fail in offline/air-gapped environments
- The `TopBar` uses `bg-secondary` for the avatar — `secondary` in tailwind.config.ts is `#b6231b` (coral-red), which is the correct Panoramex coral. This is fine but will confuse anyone reading the token name who expects "secondary" to be a subdued neutral

## Questions to Consider

- "The Dashboard is completely passive — what if the most important action for Cianya (calling back a no-response lead, confirming a reservation) were accessible with a single click from here, without navigating away?"
- "All 4 KPI cards have equal visual weight. If you had to show Cianya *one* number that told her if today is a good day or a bad day, which KPI would it be — and what if it were twice as large as the others?"
- "RecentActivity caps at 4 prospects. What if instead of 'last 4', it showed the 4 most urgent — the ones at risk of going cold or the ones with pending tasks?"
